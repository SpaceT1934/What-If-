# AGENTS.md

## 项目身份
- 项目名：**What If · 看山平行宇宙**
- 定位：基于知乎行为轨迹的认知宇宙产品
- 核心问题：**内容如何长期影响一个人的认知轨迹**

## 产品世界观
- 输入源：浏览记录、搜索行为、点赞收藏、关注变化、输出内容变化
- 输出物：认知星尘、主宇宙、平行宇宙、半山宇宙广场
- 页面语气：安静、克制、深邃，不做营销口号堆叠

## 术语白名单（必须统一）
- `星尘`（不要再用 card）
- `主宇宙`
- `半山宇宙`
- `认知轨迹`
- `平行宇宙`
- `认知切片`

## 禁用术语
- `dashboard`
- `profile card`
- `data panel`

## 视觉系统（硬约束）
- 风格：深蓝宇宙、漂浮感、glassmorphism、高级 AI 产品感
- 参考：Apple Vision Pro / OpenAI / Linear / Arc Browser
- 禁止：卡通风、NFT潮玩感、赛博朋克土味、二次元游戏 UI、后台管理感

## 动效规范（硬约束）
- 必须：缓慢、克制、呼吸感、失重漂浮感
- 建议：低频 opacity 脉冲、微弱位移、慢速粒子
- 禁止：bounce、高频闪烁、炫技特效、强跟随鼠标、快速抖动

## 页面设计原则
- 页面不是后台，是“认知宇宙空间”
- 每页至少体现：情绪感、空间感、漂浮感、留白、宇宙氛围
- 信息布局避免密集表单堆叠，优先“叙事式输入 + 生成式预览”

## 数据与内容规范
- 当前阶段：使用 **Markdown mock data**
- mock 必须模拟真实知乎行为轨迹：
  - 浏览内容
  - 搜索关键词
  - 点赞倾向
  - 收藏倾向
  - 关注变化
  - 输出内容变化
- 禁止：脱离知乎语境的纯虚构人生故事
- 重点：体现“内容 -> 认知 -> 行为”的长期影响链

## 技术栈与实现约束
- 前端：Vue3 + TailwindCSS + TypeScript + Framer Motion（或等效低侵入动效方案）
- 后端：FastAPI
- 实现要求：
  - Vue3 Composition API
  - 小组件拆分，单文件职责清晰
  - 页面视觉 token（颜色、阴影、模糊、发光）尽量复用，避免每页各写一套

## 开发状态
- 已完成：Landing Page、用户信件页、主宇宙页面
- 当前开发：新增星尘页面、Markdown 数据后端、平行宇宙详情页

## AI 执行优先级（给后续 Codex）
1. 先校准术语与世界观，再写 UI
2. 先保证气质统一，再补交互细节
3. 动效优先“弱存在感”，不抢信息内容
4. 任何新页面先判断：是否像“认知宇宙空间”，而不是传统产品页
